## 六、实施路线图

```
第 1 周（P0 核心改造）
  ├── 周一：users 表加 user_type 字段 + Alembic 迁移
  ├── 周二：注册页加类型选择 + 病人自动创建 patient_profiles
  ├── 周三：patient_profiles + doctor_patient_relations CRUD
  ├── 周四：medical_records 表 + 医生病例 CRUD
  └── 周五：检测时关联 patient_profile_id

第 2 周（P1 核心增强）
  ├── 周一：LLM 病史感知分析（拉取历史病例 + 历史检测 → prompt）
  ├── 周二：HistoryPage 前端实现（列表 + 详情 + 筛选）
  ├── 周三：DashboardPage 前端实现（ECharts 图表）
  ├── 周四：检测报告生成 + PDF 导出
  └── 周五：联调测试 + bug 修复

第 3 周（P1 收尾 + P2 起步）
  ├── cxr_images 自动记录 + 病例附件上传
  ├── 权限细化（doctor 只能操作所管 patient）
  ├── 操作日志完善
  └── P2 准备工作（医学知识库调研）
```

---

> **与 v2.0 的核心区别**：v2.0 聚焦"对话驱动的多 Agent 平台"，v3.0 回归实际——先把**医患协同 + 病史感知分析**做好，多 Agent 预留数据库但不过早投入。
