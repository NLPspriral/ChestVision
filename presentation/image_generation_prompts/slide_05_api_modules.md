# Slide 5 Backend API Modules Prompt

Create a 16:9 Chinese engineering overview slide titled "后端 API 与功能模块". This slide is a display-only technical supplement, so it should show the functional coverage clearly without requiring detailed narration.

Use a structured module matrix with five horizontal rows or five cards:
- "认证与权限": 注册 / 登录 / JWT / 角色权限
- "检测服务": 单图检测 / 批量检测 / ZIP 检测 / 检测历史
- "对话服务": SSE 流式输出 / 多 Agent 调度 / 报告生成
- "医患业务": 患者档案 / 医患关系 / 病例管理 / 医生推荐审核
- "模型业务": 数据集上传 / 远程训练 / 模型版本 / 默认模型切换

Add a subtle bottom pipeline: "前端页面 → API 路由 → 服务层 → 数据库 / AI 引擎". Use compact icons if helpful, but keep all labels readable in Chinese. The visual should communicate that the backend is not a single model-call endpoint; it coordinates authentication, detection, chat, medical collaboration, and model lifecycle management.

Style: consistent with the deck, light medical-tech background, blue-green primary accents, warm orange only for highlights. Avoid dense code blocks, raw endpoint paths, database dumps, or decorative cloud imagery.
