## 一、项目背景与目标

### 1.1 项目背景

胸部 X 光（Chest X-Ray）是全球最常用的医学影像检查手段。放射科医生每天需要阅读大量胸片，工作强度大。本系统旨在构建一个**医患协同的 AI 辅助诊断平台**，支持三种角色（管理员/医生/病人）各司其职，通过 AI 检测 + 病史综合分析，提升诊断效率和准确性。

### 1.2 项目目标

| 目标 | 说明 | 对应数据表 |
|------|------|-----------|
| **医患协同** | 三种用户类型：管理员/医生/病人，注册时选择，各有权责 | `users.user_type` |
| **病人自助** | 病人可注册登录、上传个人胸片、查看自己的检测结果和报告 | `patient_profiles` |
| **医生管理** | 医生可管理分配的病人、编辑病例病史、查看检测结果、生成报告 | `doctor_patient_relations` |
| **病史感知 AI** | LLM 分析时自动引用患者历史病例 + 历史检测结果，综合判断 | `medical_records` + `detection_tasks.referenced_record_ids` |
| **病灶检测** | YOLOv11 自动识别并框出 10 种常见胸部病变 | `detection_results` |
| **结构化病例** | 医生可创建/编辑结构化病例（主诉/现病史/既往史/诊断等） | `medical_records` |
| **智能报告** | 检测 + 病史 + 分析 → 一键生成诊断报告 | `detection_reports` |
| **预留扩展** | 数据库已预留多 Agent 管理、RAG 知识库、模型训练等扩展能力 | `agent_registry` / `agent_teams` |

## 二、核心设计理念

### 2.1 设计哲学

```
传统 PACS 系统：医生学习复杂菜单 → 找到功能 → 操作 → 看结果
本系统 v3.0：  注册选角色 → 病人上传 / 医生管理 → AI 综合病史分析 → 报告
```

### 2.2 三种角色闭环

```
┌──────────┐    分配医患关系     ┌──────────┐
│  管理员   │ ─────────────────→ │   医生    │
│  admin   │                    │  doctor   │
└──────────┘                    └────┬─────┘
       │                             │
       │ 管理系统                    │ 编辑病例、查看检测、
       │ 分配医患                    │ 综合病史分析、生成报告
       │                             │
       │                    ┌────────┴─────┐
       └──────────────────→ │    病人      │
                             │   patient    │
                             └──────────────┘
                                   │
                                   │ 上传个人胸片
                                   │ 查看自己的结果和报告
                                   │ 查看自己的病例
```

### 2.3 病史感知分析流程

```
病人上传胸片
      │
      ▼
YOLOv11 病灶检测 ────→ 检测结果
      │
      ▼
LLM 智能分析
  ├── ① 读取本次检测结果
  ├── ② 拉取该患者历史病例（medical_records）
  ├── ③ 拉取该患者历史检测结果（detection_tasks + detection_results）
  ├── ④ 对比分析：病灶大小变化？新发病灶？好转？
  └── ⑤ 综合输出：评估报告 + 风险等级 + 随访建议
```

## 七、非功能需求

### 7.1 性能要求

| 指标 | 要求 |
|------|------|
| 单张胸片检测（GPU） | ≤ 3 秒 |
| 单张胸片检测（CPU） | ≤ 10 秒 |
| LLM 病史分析响应 | ≤ 15 秒 |
| 并发用户 | ≥ 20 |

### 7.2 安全要求

| 要求 | 说明 |
|------|------|
| 认证 | JWT Token |
| 权限 | RBAC（roles + permissions）+ user_type |
| 患者隐私 | 姓名可选加密、身份证号仅存哈希、patient_code 脱敏编号 |
| 数据隔离 | 病人只能看自己的数据；医生只能看所管病人的数据 |
| 操作审计 | `operation_logs` 记录所有关键操作 |

### 7.3 数据要求

| 要求 | 说明 |
|------|------|
| 数据库 | PostgreSQL 15 + Pgvector |
| 对象存储 | MinIO（胸片原图、标注图、报告 PDF） |
| 缓存 | Redis（会话、对话历史） |
| 向量存储 | Pgvector（RAG 知识库，远期） |

## 八、数据集说明

### 8.1 检测类别（10 种胸部病变）

| class_id | 英文名 | 中文名 | 临床紧急程度 |
|----------|--------|--------|-------------|
| 0 | Nodule | 肺结节 | 中（需随访） |
| 1 | Mass | 肿块 | 高 |
| 2 | Consolidation | 肺实变 | 中 |
| 3 | Effusion | 胸腔积液 | 中 |
| 4 | Atelectasis | 肺不张 | 中 |
| 5 | Pneumothorax | 气胸 | **急症** |
| 6 | Emphysema | 肺气肿 | 低（慢性） |
| 7 | Fibrosis | 纤维化 | 低（慢性） |
| 8 | Calcification | 钙化 | 低（良性） |
| 9 | Fracture | 骨折 | 高 |

### 8.2 模型

| 模型 | 路径 | 说明 |
|------|------|------|
| YOLOv11n | `backend/models/best.pt` | 当前使用 |
| YOLOv11x | `yolo11x_train/weights/` | 大模型版本 |
